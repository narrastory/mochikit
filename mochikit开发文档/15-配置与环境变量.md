# 15 - 配置与环境变量

本章你将学会：用环境变量配置 MochiKit，以及在测试/生产环境灵活部署。

## 1. 全部环境变量

在 `.env` 里配置（或直接在系统环境设置）：

| 变量 | 用途 | 默认 |
|---|---|---|
| `BASE_URL` | LLM 端点 | `https://open.bigmodel.cn/api/anthropic` |
| `API_KEY` | LLM 密钥 | （必填） |
| `MODEL` | 默认模型名 | `glm-4.7` |
| `MOCHIKIT_WEB_API_KEY` | Web 工具密钥 | 同 `API_KEY` |
| `MOCHIKIT_RUN_INTEGRATION` | 是否跑真实 LLM 集成测试 | `0` |

`.env.example` 是模板，复制成 `.env` 填值即可。

## 2. loadConfig()

```ts
import { loadConfig } from 'mochikit';

const cfg = loadConfig();
// { baseUrl, apiKey, model, webApiKey, runIntegration }
```

行为：

- 自动加载 `.env`（从当前目录向上查找，最多 6 层）。
- 用 `override: true`，确保你的 `.env` 覆盖宿主预设（如 Vite 的 `BASE_URL='/'`）。
- 结果缓存，重复调用零开销。

`resetConfigCache()` 可清缓存（测试用）。

## 3. 指定 .env 路径

```ts
loadConfig('/path/to/my.env');
```

## 4. 不用 dotenv，手动构造

你也可以完全跳过环境变量，直接传参：

```ts
import { AnthropicAdapter } from 'mochikit';

const llm = new AnthropicAdapter({
  apiKey: process.env.MY_KEY!,
  baseURL: 'https://open.bigmodel.cn/api/anthropic',
});
```

## 5. 集成测试开关

集成测试默认 skip，避免无凭证时失败。开启：

```bash
MOCHIKIT_RUN_INTEGRATION=1 npx vitest run tests/integration
```

或 `npm run test:integration`（已内置该 env）。

## 6. 多环境建议

- **本地开发**：`.env` 填测试 key，`MOCHIKIT_RUN_INTEGRATION=1`。
- **CI/生产**：用 CI 的 secret 注入 `API_KEY` 等，`.env` 不入库（已在 `.gitignore`）。
- **不同模型**：每个 Agent 可单独设 `model` / `fallbackModel`，不必全局统一。
