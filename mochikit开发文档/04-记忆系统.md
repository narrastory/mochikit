# 04 - 记忆系统

本章你将学会：让 Agent 记住事实、按需召回、跨会话保留信息。

## 1. 为什么需要记忆

默认情况下，Agent 只记得当前对话历史（`reset()` 后就忘了）。如果你希望 Agent
“记住用户偏好”“记住项目约束”“记住过去的反馈”，就需要**记忆系统**。

MochiKit 提供统一的 `Memory` 接口，内置一个基于 Markdown 文件的实现 `MarkdownMemory`。

## 2. 创建 MarkdownMemory

```ts
import { MarkdownMemory } from 'mochikit';

const memory = new MarkdownMemory({
  dir: './.mochikit/memory', // 存储目录
});
```

每条记忆会存为一个 `.md` 文件（带 YAML frontmatter），并自动维护 `MEMORY.md` 索引。

## 3. 直接操作记忆

```ts
// 写入
const entry = await memory.add({
  name: '用户偏好简洁',
  type: 'feedback',            // user | feedback | project | reference
  description: '回答尽量短',
  body: '用户喜欢简洁直接的回答，不要长篇大论。',
});
console.log(entry.id);        // 自动生成的 id

// 列表
const all = await memory.list();

// 按关键词召回
const hits = await memory.query('回答风格');
console.log(hits.map(h => h.name));

// 读取单条
const one = await memory.get(entry.id);

// 更新
await memory.update(entry.id, { body: '新内容' });

// 删除
await memory.remove(entry.id);
```

记忆类型（type）建议：

- `user`：用户是谁、角色、偏好
- `feedback`：用户给过的反馈/纠正
- `project`：项目相关的约束、目标
- `reference`：外部资源指针（链接、文档）

## 4. 让 Agent 自主读写记忆

把 `memory` 接到 Agent，并装上记忆工具，Agent 就能自己决定何时记、何时查：

```ts
import { Agent, AnthropicAdapter, loadConfig, MarkdownMemory, createMemoryTools, AllowAllResolver, PermissionManager } from 'mochikit';

const cfg = loadConfig();
const memory = new MarkdownMemory({ dir: './.mochikit/memory' });

const agent = new Agent({
  name: 'mem-agent',
  llm: new AnthropicAdapter({ apiKey: cfg.apiKey, baseURL: cfg.baseUrl }),
  model: cfg.model,
  systemPrompt: '你可以用 memory_write 记住重要事实，用 memory_read 召回。',
  memory,
  tools: createMemoryTools(memory),
  permission: new PermissionManager({ resolver: new AllowAllResolver() }),
});

// 第一轮：记住
await agent.run('请记住：我最喜欢的编程语言是 Rust。');

// 第二轮：召回
console.log(await agent.run('我最喜欢什么编程语言？用 memory_read 查一下。'));
// 预期：Rust
```

模型会调用 `memory_write` 写入，之后用 `memory_read` 召回。

## 5. 自定义召回逻辑

默认 `query` 用关键词匹配。如果你想用 LLM 做语义召回，传 `recall`：

```ts
const memory = new MarkdownMemory({
  dir: './.mochikit/memory',
  recall: async (entries, needle, k) => {
    // 这里可以调用 LLM，从 entries 里挑最相关的 k 条
    // 简单演示：按包含关系
    return entries
      .filter(e => `${e.name} ${e.body}`.includes(needle))
      .slice(0, k);
  },
});
```

## 6. 持久化

`MarkdownMemory` 直接写磁盘，进程重启后记忆依然在。存储目录建议加进 `.gitignore`。
