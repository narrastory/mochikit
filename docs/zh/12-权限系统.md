# 11 - 权限系统

本章你将学会：控制 Agent 能不能执行某个工具调用，防止危险操作。

## 1. 为什么需要权限

Agent 会调用 `bash`、`write_file` 等工具，模型可能生成危险命令（`rm -rf /`）或写错路径。
`PermissionManager` 在**每个工具执行前**做一次检查，决定放行、拒绝、还是询问。

## 2. 三段闸门

每次工具调用依次过三关：

1. **deny**：命中硬拒绝规则 → 直接阻止。
2. **rule**：命中规则产生 reason → 交给 `PermissionResolver` 决定。
3. **ask**：Resolver 决定 allow / deny。
4. 都没命中 → 默认放行。

## 3. 快速使用

```ts
import { Agent, AnthropicAdapter, loadConfig, createBashTool, PermissionManager, AllowAllResolver } from 'mochikit';

const cfg = loadConfig();

const permission = new PermissionManager({
  resolver: new AllowAllResolver(), // 升级到 ask 时默认放行（适合沙箱/测试）
});

const agent = new Agent({
  name: 'safe',
  llm: new AnthropicAdapter({ apiKey: cfg.apiKey, baseURL: cfg.baseUrl }),
  model: cfg.model,
  systemPrompt: '你是助手。',
  tools: [createBashTool()],
  permission,
});
```

不配 `permission` 时，Agent 会用一个默认的 `PermissionManager`（默认 Resolver 是 `DenyAllResolver`，
即所有 ask 都拒绝）。

## 4. 写权限规则

```ts
import { PermissionManager, AllowAllResolver } from 'mochikit';

const permission = new PermissionManager({
  resolver: new AllowAllResolver(),
  rules: [
    {
      name: 'no-rm-rf',
      tools: ['bash'], // 只对 bash 工具生效；不写 tools 则对所有工具
      check: (ctx) => {
        const cmd = String(ctx.tool.input.command);
        return /rm\s+-rf/.test(cmd) ? 'deny' : 'passthrough';
        // 'deny'   → 直接拒
        // 'ask'    → 交 Resolver
        // 'passthrough' → 让下一条规则决定
        // 返回字符串 → 当作 reason，交 Resolver
      },
      reason: 'rm -rf 是危险操作',
    },
    {
      name: 'workspace-write-only',
      tools: ['write_file', 'edit_file'],
      check: (ctx) => {
        const p = String(ctx.tool.input.path);
        return p.startsWith(process.cwd()) ? 'passthrough' : 'deny';
      },
      reason: '禁止写工作区外文件',
    },
  ],
});
```

规则检查函数 `check(ctx)` 的返回值：

| 返回 | 含义 |
|---|---|
| `'allow'` | 放行 |
| `'deny'` | 拒绝 |
| `'ask'` | 交 Resolver 决定（用 `reason` 字段） |
| `'passthrough'` 或 `null` | 交给下一条规则 |
| 任意字符串 | 当作 reason，交 Resolver |

## 5. PermissionResolver

Resolver 决定“ask”类请求的最终结果：

```ts
import { AllowAllResolver, DenyAllResolver, type PermissionResolver } from 'mochikit';

new AllowAllResolver(); // 全部放行
new DenyAllResolver();  // 全部拒绝

// 自定义：交互式询问（CLI 场景）
const interactive: PermissionResolver = {
  async resolve(ctx, reason) {
    const answer = await askUser(`${reason}\n允许执行 ${ctx.tool.name} 吗？(y/n)`);
    return answer === 'y' ? 'allow' : 'deny';
  },
};
```

## 6. 运行时调整

```ts
permission.addRule({ /* ... */ });   // 加规则
permission.setResolver(new DenyAllResolver()); // 换 Resolver
agent.registerPermissionRule({ /* ... */ });   // 通过 Agent 加规则
```

## 7. 与 Hook 的区别

- `PermissionManager`：声明式、专注“允许/拒绝工具”。
- `PreToolUse` Hook：更灵活，能改写结果、终止循环、做副作用。

权限适合“安全策略”，Hook 适合“通用扩展”。两者可叠加使用（Hook 先跑，Permission 后跑）。
