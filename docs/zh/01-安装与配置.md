# 01 - 安装与配置

本章你将学会：把 MochiKit 装到项目里、配置好 LLM 凭证、跑通第一个脚本。

## 1. 环境要求

- Node.js ≥ 18（推荐 20+）
- 一个 LLM API Key。本文以 GLM 智谱为例（兼容 Anthropic 协议）。

## 2. 安装

如果你的项目已初始化，直接装：

```bash
npm install mochikit
```

如果你是用本仓库源码（开发/学习），则在仓库根目录：

```bash
npm install
```

这会安装运行依赖 `@anthropic-ai/sdk`、`dotenv`，以及开发依赖 `typescript`、`tsx`、`vitest`。

## 3. 配置凭证（.env）

在项目根目录创建 `.env` 文件（不要提交到 git）：

```dotenv
# LLM 端点（GLM 智谱 Anthropic 兼容端点）
BASE_URL=https://open.bigmodel.cn/api/anthropic
API_KEY=你的-api-key
MODEL=glm-4.7

# Web 工具用的 key（通常与 API_KEY 相同）
MOCHIKIT_WEB_API_KEY=你的-api-key

# 设为 1 才会运行真实 LLM 集成测试
MOCHIKIT_RUN_INTEGRATION=0
```

> 提示：GLM 的 API Key 形如 `xxxxxxxx.xxxxxxxxxxxxxxxx`，在智谱开放平台获取。
> `BASE_URL`、`API_KEY`、`MODEL` 三个变量缺一不可。

### 3.1 多供应商配置（Multi-Provider）

MochiKit 支持同时配置**多个 LLM 供应商**，通过 `{NAME}_API_KEY` 命名约定自动发现：

```dotenv
# === 默认供应商（无前缀，向后兼容）===
API_KEY=你的-glm-key
BASE_URL=https://open.bigmodel.cn/api/anthropic
MODEL=glm-4.7

# === 命名供应商 ===
# DeepSeek
DEEPSEEK_API_KEY=sk-你的-deepseek-key
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-chat

# OpenAI
OPENAI_API_KEY=sk-你的-openai-key
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o
```

**前缀规则**：
- 前缀必须全部**大写**（如 `DEEPSEEK`、`OPENAI`），至少 2 个字符。
- 框架会自动扫描 `process.env` 中所有匹配 `{PREFIX}_API_KEY` 的变量，每个前缀即为一个供应商。
- 供应商名在调用 `loadConfig()` 时**不区分大小写**（`'DeepSeek'`、`'deepseek'`、`'DEEPSEEK'` 均可）。

选择特定供应商：

```ts
import { loadConfig } from 'mochikit';

const glmCfg    = loadConfig();           // 默认供应商（bare API_KEY）
const deepCfg   = loadConfig('deepseek'); // DeepSeek 供应商
const openaiCfg = loadConfig('openai');   // OpenAI 供应商
```

## 4. 在代码里读取配置

用 `loadConfig()` 一行读取：

```ts
import { loadConfig } from 'mochikit';

const cfg = loadConfig();
console.log(cfg.baseUrl); // https://open.bigmodel.cn/api/anthropic
console.log(cfg.model);   // glm-4.7
console.log(cfg.apiKey);  // 你的 key

// 查看所有已配置的供应商
console.log(Object.keys(cfg.providers)); // ['default', 'deepseek', 'openai']
console.log(cfg.defaultProvider);        // 'default'
```

`loadConfig()` 会自动加载 `.env`，并缓存结果。它返回：

| 字段 | 说明 |
|---|---|
| `baseUrl` | 当前供应商的 LLM 端点 |
| `apiKey` | 当前供应商的 LLM 密钥 |
| `model` | 当前供应商的默认模型名 |
| `webApiKey` | 当前供应商的 Web 工具密钥 |
| `runIntegration` | 是否开启集成测试 |
| `defaultProvider` | 当前供应商名称（`'default'` 或其他） |
| `providers` | 所有已发现的供应商配置映射 |

> 注意：Vite/Vitest 环境会预设 `process.env.BASE_URL='/'`，因此 MochiKit 用 `override: true`
> 加载 `.env`，确保你的配置生效。

## 5. 运行第一个脚本

创建 `hello.ts`：

```ts
import { Agent, AnthropicAdapter, loadConfig } from 'mochikit';

const cfg = loadConfig();
const agent = new Agent({
  name: 'hello',
  llm: new AnthropicAdapter({ apiKey: cfg.apiKey, baseURL: cfg.baseUrl }),
  model: cfg.model,
  systemPrompt: '你是一个简洁的助手。',
});

const answer = await agent.run('用一句话介绍 TypeScript。');
console.log(answer);
```

运行：

```bash
npx tsx hello.ts
```

预期输出：模型返回的一句话简介。

### 5.1 为不同 Agent 使用不同供应商

```ts
const defaultCfg = loadConfig();
const deepCfg    = loadConfig('deepseek');

// 管理 Agent 用 GLM
const manager = new Agent({
  name: 'manager',
  llm: new AnthropicAdapter({ apiKey: defaultCfg.apiKey, baseURL: defaultCfg.baseUrl }),
  model: defaultCfg.model,
  systemPrompt: '你是一个管理助手。',
});

// 工作 Agent 用 DeepSeek
const worker = new Agent({
  name: 'worker',
  llm: new AnthropicAdapter({ apiKey: deepCfg.apiKey, baseURL: deepCfg.baseUrl }),
  model: deepCfg.model,
  systemPrompt: '你是一个高效的执行助手。',
});
```
