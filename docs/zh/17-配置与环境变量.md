# 15 - 配置与环境变量

本章你将学会：用环境变量配置 MochiKit、多供应商设置、以及在测试/生产环境灵活部署。

## 1. 全部环境变量

在 `.env` 里配置（或直接在系统环境设置）：

### 默认供应商（无前缀）

| 变量 | 用途 | 默认 |
|---|---|---|
| `BASE_URL` | 默认 LLM 端点 | `https://open.bigmodel.cn/api/anthropic` |
| `API_KEY` | 默认 LLM 密钥 | （必填） |
| `MODEL` | 默认模型名 | `glm-4.7` |
| `MOCHIKIT_WEB_API_KEY` | Web 工具密钥（默认） | 同 `API_KEY` |
| `MOCHIKIT_RUN_INTEGRATION` | 是否跑真实 LLM 集成测试 | `0` |

### 命名供应商（前缀约定）

格式：`{NAME}_API_KEY`、`{NAME}_BASE_URL`、`{NAME}_MODEL`、`{NAME}_WEB_API_KEY`

| 变量模式 | 用途 | 默认 |
|---|---|---|
| `{NAME}_API_KEY` | 供应商 LLM 密钥 | — |
| `{NAME}_BASE_URL` | 供应商 LLM 端点 | `''`（空） |
| `{NAME}_MODEL` | 供应商默认模型 | `''`（空） |
| `{NAME}_WEB_API_KEY` | 供应商 Web 工具密钥 | 同 `{NAME}_API_KEY` |

**前缀规则**：
- 前缀必须全部**大写**，至少 2 个字符，只含大写字母和数字（正则：`/^[A-Z][A-Z0-9]+_API_KEY$/`）。
- 框架启动时自动扫描 `process.env`，发现所有匹配的前缀。
- 如果 `{NAME}_API_KEY` 为空，该供应商会被跳过。

示例 `.env`：

```dotenv
# 默认供应商（GLM）
BASE_URL=https://open.bigmodel.cn/api/anthropic
API_KEY=your-glm-key
MODEL=glm-4.7

# DeepSeek
DEEPSEEK_API_KEY=sk-your-deepseek-key
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-chat

# OpenAI
OPENAI_API_KEY=sk-your-openai-key
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o
```

`.env.example` 是模板，复制成 `.env` 填值即可。

## 2. loadConfig()

```ts
import { loadConfig } from 'mochikit';

// 默认供应商
const cfg = loadConfig();

// 指定供应商
const deepCfg = loadConfig('deepseek');
const openaiCfg = loadConfig('openai');
```

### 返回值：`MochiConfig`

```ts
interface MochiConfig {
  // 扁平字段 — 反映当前选中的供应商（向后兼容）
  baseUrl: string;
  apiKey: string;
  model: string;
  webApiKey: string;
  runIntegration: boolean;

  // 多供应商字段（v0.2.0 新增）
  defaultProvider: string;                    // 当前供应商名称
  providers: Record<string, ProviderConfig>;  // 所有已发现的供应商
}

interface ProviderConfig {
  apiKey: string;
  baseUrl: string;
  model: string;
  webApiKey: string;
}
```

`providers` 中始终包含 `'default'` 键（对应无前缀的默认供应商）。

### 调用形式

| 调用 | 供应商 | .env 路径 |
|---|---|---|
| `loadConfig()` | `'default'` | 自动查找 |
| `loadConfig('deepseek')` | `'deepseek'` | 自动查找 |
| `loadConfig('/path/to/.env')` | `'default'` | 显式路径 |
| `loadConfig('deepseek', '/path/to/.env')` | `'deepseek'` | 显式路径 |

### 供应商名与路径的自动区分

`loadConfig` 的第一个参数既可以是供应商名，也可以是 `.env` 文件路径。框架通过以下规则自动区分：
- 包含 `/` 或 `\`（路径分隔符）→ 视为路径
- 以 `.env` 结尾 → 视为路径
- 其他 → 视为供应商名（不区分大小写）

### 未知供应商

若指定了未配置的供应商名，框架会抛出明确的错误：

```
Unknown provider "unknown". Available providers: default, deepseek.
Set UNKNOWN_API_KEY, UNKNOWN_BASE_URL, UNKNOWN_MODEL in your .env file.
```

### 缓存行为

- 首次调用 `loadConfig()` 时加载 `.env` 并扫描所有供应商，结果缓存。
- 后续调用（无论传入什么供应商名）直接从缓存构建返回值，零开销。
- `resetConfigCache()` 可清缓存（测试用）。

## 3. 指定 .env 路径

```ts
// 使用自定义 .env 文件 + 默认供应商
loadConfig('/path/to/my.env');

// 指定供应商 + 自定义 .env 文件
loadConfig('openai', '/path/to/production.env');
```

## 4. 不用 dotenv，手动构造

你也可以完全跳过环境变量，直接传参：

```ts
import { AnthropicAdapter } from 'mochikit';

const llm = new AnthropicAdapter({
  apiKey: process.env.MY_KEY!,
  baseURL: 'https://open.bigmodel.cn/api/anthropic',
});
```

## 5. 集成测试开关

集成测试默认 skip，避免无凭证时失败。开启：

```bash
MOCHIKIT_RUN_INTEGRATION=1 npx vitest run tests/integration
```

或 `npm run test:integration`（已内置该 env）。

## 6. 多环境建议

- **本地开发**：`.env` 填测试 key，`MOCHIKIT_RUN_INTEGRATION=1`。
- **CI/生产**：用 CI 的 secret 注入 `API_KEY` 等，`.env` 不入库（已在 `.gitignore`）。
- **不同模型**：每个 Agent 可单独设 `model` / `fallbackModel`，不必全局统一。
- **多供应商**：
  - 方案 A：所有供应商的 key 放在同一个 `.env`，通过 `loadConfig('provider')` 切换。
  - 方案 B：每个供应商独立的 `.env` 文件，通过 `loadConfig('provider', '/path/.env')` 加载。
  - 方案 C：CI 注入环境变量（不依赖 `.env` 文件），`loadConfig()` 自动从 `process.env` 发现。

## 7. 典型场景

### 7.1 Manager 用便宜模型，Worker 用强模型

```ts
const cheapCfg = loadConfig('deepseek');   // DeepSeek 便宜
const strongCfg = loadConfig('openai');    // OpenAI 能力强

const manager = new Agent({
  name: 'manager',
  llm: new AnthropicAdapter({ apiKey: cheapCfg.apiKey, baseURL: cheapCfg.baseUrl }),
  model: 'deepseek-chat',
  systemPrompt: '你负责分解任务。',
});

const worker = new Agent({
  name: 'worker',
  llm: new AnthropicAdapter({ apiKey: strongCfg.apiKey, baseURL: strongCfg.baseUrl }),
  model: 'gpt-4o',
  systemPrompt: '你负责执行复杂推理。',
});
```

### 7.2 查看所有可用供应商

```ts
const cfg = loadConfig();
for (const [name, p] of Object.entries(cfg.providers)) {
  console.log(`${name}: ${p.model} @ ${p.baseUrl}`);
}
// default: glm-4.7 @ https://open.bigmodel.cn/api/anthropic
// deepseek: deepseek-chat @ https://api.deepseek.com/v1
// openai: gpt-4o @ https://api.openai.com/v1
```
