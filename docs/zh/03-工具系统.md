# 03 - 工具系统

本章你将学会：使用内置工具、自己写一个工具、把工具注册给 Agent。

工具（Tool）是 Agent “动手做事”的能力：读文件、跑命令、查数据库、调 API……
模型决定调用哪个工具、传什么参数，框架负责执行并把结果喂回模型。

## 1. 内置工具一览

MochiKit 开箱即用：

| 工厂函数 | 工具 | 说明 |
|---|---|---|
| `createBashTool()` | `bash` | 执行 shell 命令 |
| `createFsTools()` | `read_file` / `write_file` / `edit_file` / `glob` / `grep` | 文件读写与搜索 |
| `createWebSearchTool(key)` | `web_search` | GLM 网络搜索（见第 14 章） |
| `createWebReaderTool(key)` | `web_reader` | GLM 网页阅读（见第 14 章） |
| `createMemoryTools(memory)` | `memory_read` / `memory_write` | 读写记忆（见第 4 章） |
| `createTaskTools(tasks)` | `create_task` / `claim_task` / `complete_task` | 任务管理（见第 9 章） |
| `createTeamTools(bus, name)` | `send_message` / `check_inbox` | 团队通信（见第 8 章） |

## 2. 给 Agent 装上内置工具

```ts
import { Agent, AnthropicAdapter, loadConfig, createBashTool, createFsTools, AllowAllResolver, PermissionManager } from 'mochikit';

const cfg = loadConfig();
const agent = new Agent({
  name: 'dev',
  llm: new AnthropicAdapter({ apiKey: cfg.apiKey, baseURL: cfg.baseUrl }),
  model: cfg.model,
  systemPrompt: '你是开发助手，善用工具。',
  tools: [createBashTool(), ...createFsTools()],
  permission: new PermissionManager({ resolver: new AllowAllResolver() }),
});

console.log(await agent.run('读取 package.json 并告诉我项目名字'));
```

工具会以 JSON Schema 形式自动暴露给模型，无需手动声明。

## 3. 自定义工具（推荐方式：继承 BaseTool）

```ts
import { BaseTool, type ToolContext } from 'mochikit';

class WeatherTool extends BaseTool {
  // 1. 声明工具定义（名字、描述、参数 schema）
  readonly definition = {
    name: 'get_weather',
    description: '查询某城市的天气。',
    input_schema: {
      type: 'object',
      properties: {
        city: { type: 'string', description: '城市名' },
      },
      required: ['city'],
    },
  };

  // 2. 实现执行逻辑
  async execute(input: Record<string, unknown>, _ctx: ToolContext): Promise<string> {
    const city = this.requireString(input, 'city'); // 类型安全的取参
    // 这里可以调真实天气 API
    return `${city} 今天晴，25℃`;
  }
}

const agent = new Agent({
  // ...
  tools: [new WeatherTool()],
});
```

`BaseTool` 提供了取参小工具：

- `this.requireString(input, 'city')`：必填字符串，缺失抛错
- `this.optionalString(input, 'note')`：可选字符串
- `this.optionalNumber(input, 'count')`：可选数字

## 4. 快速包装：toolFromFunction

如果工具逻辑很简单，不想建类，可以用 `toolFromFunction`：

```ts
import { toolFromFunction } from 'mochikit';

const echo = toolFromFunction(
  {
    name: 'echo',
    description: '原样返回输入文本。',
    input_schema: { type: 'object', properties: { text: { type: 'string' } }, required: ['text'] },
  },
  async (input) => `echo: ${input.text}`,
);

const agent = new Agent({ /* ... */ tools: [echo] });
```

## 5. 工具上下文 ToolContext

每个工具执行时会收到一个上下文对象：

```ts
interface ToolContext {
  agentName: string;      // 当前 Agent 名
  cwd: string;            // 工作目录
  memory?: Memory;        // Agent 的记忆（如配置了）
  bus?: MessageBus;       // 消息总线（如配置了）
  tasks?: TaskStore;      // 任务存储（如配置了）
  runtime?: Record<string, unknown>; // 运行时附加数据
}
```

工具里可以读 `ctx.memory` / `ctx.tasks` 等来与 Agent 的基础设施联动。

## 6. 动态注册 / 移除工具

运行中也能加工具：

```ts
agent.registerTool(new WeatherTool());
```

## 7. 命名空间隔离（MCP 风格）

当你接入了外部工具集（比如某个 MCP 服务器），可以用命名空间避免名字冲突：

```ts
import { ToolRegistry } from 'mochikit';
// Agent 内部已有 registry；这里演示独立使用
const reg = new ToolRegistry();
reg.registerNamespaced('github', new WeatherTool());
// 实际工具名变成 github__get_weather
```

## 8. 并发安全标记

如果某工具可安全并发执行（比如只读查询），重写 `isConcurrencySafe`：

```ts
class SafeQueryTool extends BaseTool {
  // ...
  isConcurrencySafe(): boolean {
    return true;
  }
}
```

框架可据此把多个安全工具批量并发执行（当前版本为顺序派发，预留扩展）。

## TodoWrite 工具（对话内规划）

`todo_write` 让模型在长任务中先列计划再执行。不同于 TaskStore（持久化 DAG），它是当前对话内的轻量规划。

```ts
import { createTodoWriteTool } from 'mochikit';

const agent = new Agent({
  // ...
  tools: [createTodoWriteTool()],
});
```

模型会在复杂任务开始时自动调用 `todo_write` 列出步骤，做完一项标记一项。如果连续 3 轮没更新，系统会自动提醒。

工具接收 `{todos: [{content: string, status: 'pending'|'in_progress'|'completed'}]}`，每次调用会全量替换列表。

## Bash 后台执行

`bash` 工具现在支持 `run_in_background: true` 参数。用于 `npm install`、`npm test` 等长时间命令时，Agent 不会阻塞等待。

```ts
import { createBashTool, BackgroundTaskManager } from 'mochikit';

const agent = new Agent({
  // ...
  tools: [createBashTool()],
  backgroundTasks: new BackgroundTaskManager(),
});
```

模型判断哪些命令适合后台执行（安装、构建、测试等），Agent 会立即收到占位结果继续工作。后台命令完成后会自动注入通知。
